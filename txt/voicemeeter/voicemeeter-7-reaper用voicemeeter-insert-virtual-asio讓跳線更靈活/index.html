voicemeeter-7 reaper用Voicemeeter Insert Virtual ASIO，讓跳線更靈活

在reaper的ASIO驅動裡，除了Voicemeeter Virtual ASIO和Voicemeeter AUX Virtual ASIO，還有Voicemeeter Insert Virtual ASIO可供我們選擇，
前面的文章有提到用Voicemeeter Virtual ASIO來把b1的訊號做處理送到v1，這是比較普通的用法，
而這篇會介紹的Voicemeeter Insert Virtual ASIO更加強大，可以自由選擇從h1、h2、h3、v1、v2來接訊號做處理，也可以任意發送到h1、h2、h3、v1、v2，
訊號在經過處理後才會發回h或v，最後才會依照你所設定的跳線參數發送到a做監聽或發到b給其他軟體使用。

從設定檔開啟Voicemeeter Insert Virtual ASIO的通道

在從reaper使用Voicemeeter Insert Virtual ASIO這個虛擬ASIO驅動前，要先從設定檔開啟Voicemeeter Insert Virtual ASIO的通道，才能讓reaper順利的接管這些通道的訊號。

  1. 找到設定檔，按快顯見，找編輯enter
  2. 打開搜尋框，輸入
    Insert
    上下動一下會看到這一行
  3. 把'(單引號)裡面的數字改為1，表示啟用，另外c15和c16保持0即可，改完會像這樣
  4. 按ctrl+s存檔，修改完成
  5. 接下來按照之前說過的方法將設定檔載入voicemeeter，載入方法可查看
    點我查看voicemeeter-5 跳線規則的設定

Voicemeeter Insert Virtual ASIO的通道規則

在Voicemeeter Insert Virtual ASIO會有22個通道，
1、2通道是h1、3、4通道是h2，5、6通道是h3，
7到14通道是v1，最後的15到22通道是v2，
要注意的是h1雖然有兩個通道，但這兩個通道並不是一左一右，
第一通道是左、右雙聲道，正常的立體聲，第二通到則是右、左雙聲道，顛倒的立體聲，後面的h跟v也是如此。

補充

我們知道，c15和c16是v2的通道
在設定檔中c15和c16之所以保持為0，是因為所有改為1的通道在reaper使用Voicemeeter Insert Virtual ASIO之後都會被reaper接管，
在被接管的狀態下，就算設定檔裡v2有開啟a，但我們將聲音從v2輸出都是聽不到聲音的，
因此將c15和c16解放出來的方法就是不將其參數改為1，這樣就不會被Voicemeeter Insert Virtual ASIO接管了
此時如果v2有開啟a，我們將系統或軟體的輸出裝置設為v2就能聽到聲音囉。

從reaper指定通道的概念

在reaper裡我們可以輕鬆的選擇輸入通道和輸出通道，不管是普通的軌道或主控軌都可以單獨設定，
這就給我們很大的彈性，比方說
我將軌道的輸入通道以第一通道輸入，輸出通道用7通道，
此時我的跳線設定只有v1開啟a1，路線會像這樣
麥克風→h1→reaper第一通道輸入→reaper處理→reaper第七通道輸出→v1→a1→聽到聲音
當然，你也可以隨時改變輸出輸入的通道，如此就不用每次都要修改設定檔來達到不同的目的。

reaper上的Voicemeeter Insert Virtual ASIO通道設定

要在reaper上使用Voicemeeter Insert Virtual ASIO，要先將ASIO驅動調成Voicemeeter Insert Virtual ASIO，以及設定通道。

  1. 打開reaper
  2. 按
    ctrl+p 打開首選項
  3. 在樹狀檢視找到設備
  4. tab找
    內容頁 ASIO驅動: 輸出範圍: 首先 最後 首先 最後 下拉式方塊
  5. 上下找
    Voicemeeter Insert Virtual ASIO
  6. tab將接下來的下拉方塊調成這樣
    下拉式方塊 1: IN#1 Left 折疊
    下拉式方塊 22: IN#5 BR 折疊
    下拉式方塊 1: IN#1 Left 折疊
    下拉式方塊 22: IN#5 BR 折疊
    這樣調的目的是讓reaper接管輸入、輸出的1到22通道
  7. tab找確定enter，完成

如何指定輸出和輸入通道

在說明之前，要先了解指定輸出、輸入有兩種選擇，
第一種是指定主控軌的輸出和輸入通道，如此接下來新增的軌道就不用再指定輸出輸入通道，
第二種是對普通軌道指定輸出輸入通道，不過每新增一條軌道，都要重新指定一次。

普通軌道指定輸入通道的方法

先說普通軌道的指定方法:

  1. 打開reaper
  2. 新增一條軌道
  3. 找到新增的軌道，按快顯鍵，找
    輸入: 單聲道 勾選 子功能表
    往右展開，看到
    IN#1 Left 勾選 i
    p.s 因為是單聲道輸入，所以一個選項只有一個left(左)或right(右)
    IN#1是通道編號，從單聲道輸入的功能表裡會有四個IN#1
  4. 往下可以選擇h2或h3，分別是
    IN#3、IN#4(h2的通道)
    IN#5、IN#6(h3)
  5. 另外也能用立體聲來輸入，不過你的麥克風要有立體聲才行，或是用線路輸入的訊號才要用立體聲輸入，
    在找到軌道後按快顯鍵，往下找
    輸入: 立體聲 子功能表
    往右展開，會看到
    IN#1 Left / IN#1 Right i
    以及顛倒的立體聲
    IN#1 Right / IN#2 Left i
    這兩個都是h1，接下來
    IN#2 Left / IN#2 Right i
    IN#2 Right / IN#3 Left i
    則是h2的正常立體聲跟顛倒的立體聲，以此類推

普通軌道指定輸出通道的方法

  1. 步驟跟上面一樣，先找到軌道，接著按
    alt+快顯鍵 找
    音頻硬件輸出 子功能表
    p.s 如果你叫出的選單跟單按快顯鍵一樣，那就把alt鍵按九一些再按快顯鍵
  2. 往右展開，看到
    1: IN#1 Left / IN#1 Right
    看起來就跟前面選擇立體聲輸入是相同的選單，
    一般我會直接再這邊往下找
    7: IN#4 Left / IN#4 Right 7
    這個通道對應到v1
  3. 在跳線規則裡只要把v1打開a1跟b1就能聽到聲音，同時把聲音送到b1，
    當然h1也必須要對應到硬體裝置，用麥克風輸入，v1這邊才會有訊號，詳細的說明可以回頭參考前幾篇的說明

p.s 前面我們有把c15、c16保持不開啟，所以15通道之後的通道基本上在reaper裡也用不了，
以我自己的設定來說v2是拿來監聽的，所以v2除了打開a之外，b都不會再進行發送的動作。

補充

另外還有另一種指定硬體監聽設備的方法，可以用再給主控軌設定輸出通道，也可以用在普通軌的輸出設定，
首先，要先顯示主控軌，按
ctrl+alt+m 可顯示主控軌
接著在主控軌或普通軌上按
i 打開軌道路由視窗
裡面有一個
音頻硬件輸出 下拉方塊
用上、下就可以選擇要硬件輸出的通道，
不過這個下拉框很難用，選到某條通道後過一兩秒不動就會默認選擇你停的通道為新的硬件輸出，
所以不想折騰的朋友，還是從普通軌透過快顯鍵來選擇輸入、輸出通道，我覺得會比較方便。
